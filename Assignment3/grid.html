Layout Options using CSS

Positioning

CSS allows us to position elements on a page wherever we want by either relative or absolute positioning.

Relative

position: relative; allows us to position the element relative to where the element would normally appear.

Absolute

If we need to place an element in a specific place, regardless of where it normally would appear, absolute positioning will pull the element out of the normal calculation and place the element anywhere on the page (relative to the page!)

position: absolute;

Inline-Block

Inline-block gives us the best of both worlds (display: inline , display: block). Normally, with display: inline; we aren't able to set margin and width, however with display: inline-block; we can! I basically default to display: inline-block; anytime I want items that are normally displayed as block elements, to line up next to each other.

CSS Columns

Admittedly, this is the layout feature I use the least. I find myself kind of foregetting about this feature because I am usually in need of a more robust solution to creating columns (see: Grid). By declaring column-count: 4; we can get a very basic four column layout (not all that different from a newspaper)

Flexbox & Grid

Both of these layout options are the modern standard in web development today. Typically I will use both of these layout options together (like on this page!).

Flexbox

display: flex; must be declared on the parent element in order to activate flex, which works for parent/child relationships only. Grandchild elements will not be affected by the grandparent declaration of display: flex;. Flexbox gives us a quick way to lay boxes out in a row (and column, although not as well).

Grid

My absolute favorite way to lay out a web page. display: grid; gets us started by declaring on the parent element. Similar to flex, only the child element will be affected by the template design (although every element inside the affected child will be inside the grid). This page makes use of Grid to extend the left navbar straight down the page without much code at all.

Which is best?

Without question, Grid is the most powerful and IMO requires the least amount of code (or in my case, thinking :p). I like to think of Grid as the macro-design scheme whereas Flex is used inside of the 'master' elements to easily arrange elements in a row - not to mention very easy centering features! justify-content: center; and align-items: center; declared on the parent element will horizontally and vertically center any item. Anytime I don't have to write margin: 0 auto; is a good day!

If I had to choose one over the other, I would choose Grid for the simple fact that I can create my layout template before I add any content to the page.
